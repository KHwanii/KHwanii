1. 관리자용 웹사이트(NoirozeServer)
    1) 동작 원리
        - 서버에 로그인 요청(post) 및 로그인 검증 처리 
        - api를 통해 인증이 된다면 서버로 검증완료+토큰(auth-token) 전달
        - 전달된 토큰은 로컬에 저장
        - auth-token 존재해야(로그인 필수) 사이트에 접근 가능

    2) 주요 기능
        - 대시보드 : 동별 평균 데시벨, 동별 소음 발생 횟수 비율 그래프 제공
        - 게시판 : 민원접수 게시판, 커뮤니티 게시판  
            -> 민원접수 게시판 : 앱에서 민원접수 등록 -> api서버 -> 웹 페이지에서 확인 및 답변 가능
            -> 커뮤니티 게시판 : 앱에서 커뮤니티 게시글 등록 -> api서버 -> 내용 확인만 가능
        - 소음 확인 : 각 동에 대한 월별 평균 데시벨, 일별 평균 데시벨, 호수별 평균 데시벨 그래프 제공

    3) 특이사항 및 개선사항
        - 페이지 제작 중 사용한 테스트 코드 및 템플릿은 주석처리 하거나 남겨둠
        - 각 동에 대한 데시벨 데이터 요청시 (1040, 'Too many connections') 오류 발생.

2. 사용자용 어플리케이션(myapp)
    1) 동작 원리
        - API를 통해 소음수집 위치 등의 정보 수신
             - 토큰기반 인증을 통한 로그인 // 로그인 정보를 바탕으로 해당 사용자의 정보만 필터링
    2) 주요 기능
	- 현재 우리집에서 발생하는 소음의 크기를 지수로 변환하여 메인화면에 표현
	- 시간대, 소음종류 등의 기준에 따라 현재 소음상태를 이미지로 제공
	- 동별, 월별 소음크기를 그래프로 제공하여 소음의 변화를 쉽게 관찰가능하도록 제공
	- 윗집에서 우리집으로 내려오는 소음의 경우, 층간소음 기준치를 넘는 소음이 측정 시
	  블랙박스처럼 녹음파일 확인 가능
	- 여러가지 게시판과 민원 접수 등의 기능을 활용하여 주민들과 소통, 쉬운 민원접수 등의
	   활동이 가능
    3) 개선사항
        - 함수들의 ViewModel 분리 필요.
        - 코루틴 적용하여 데이터를 비동기적으로 불러오도록 코드 수정 필요
